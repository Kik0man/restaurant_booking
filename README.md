# Ресторан "Вкус жизни" – система бронирования столиков

Проект представляет собой веб-приложение на Django для бронирования столиков в ресторане. Реализованы главная страница с описанием и обратной связью, страница "О ресторане", форма бронирования, личный кабинет с историей и управлением бронями, а также админ-панель для управления всеми данными. Используются PostgreSQL, Bootstrap, Docker и Docker Compose.

## Структура проекта

- `config/` – настройки проекта (settings.py, urls.py)
- `main/` – приложение для главной страницы, контента (SiteContent) и обратной связи (ContactMessage)
- `bookings/` – модели столиков (Table) и бронирований (Booking), форма бронирования
- `accounts/` – расширение пользователя (Profile), регистрация, личный кабинет, управление бронями
- `templates/` – HTML-шаблоны (base.html, main/, bookings/, accounts/, registration/)
- `static/` – статические файлы (CSS, изображения)
- `media/` – пользовательские файлы (загруженные через админку изображения)
- `docker-compose.yml`, `Dockerfile` – для контейнеризации
- `.env` – переменные окружения (секретный ключ, параметры БД, DEBUG)

## Технологии

- Python 3.10, Django 4.2+
- PostgreSQL 15
- Bootstrap 5 + иконки Bootstrap Icons
- Docker и Docker Compose
- Git, соблюдение PEP8

## Установка и запуск (через Docker)

1. Клонируйте репозиторий:
   `git clone <url-репозитория> && cd restaurant_booking`

2. Скопируйте `.env.example` в `.env` (или создайте `.env` вручную) со следующим содержимым:
SECRET_KEY=ваш-секретный-ключ
DEBUG=1
DB_NAME=restaurant_db
DB_USER=restaurant_user
DB_PASSWORD=secure_password
DB_HOST=db
DB_PORT=5432

3. Запустите контейнеры (убедитесь, что Docker Desktop запущен):
`docker-compose up -d`

4. Выполните миграции:
`docker-compose exec web python manage.py migrate`

5. Создайте суперпользователя для доступа в админку:
`docker-compose exec web python manage.py createsuperuser`

6. Соберите статические файлы:
`docker-compose exec web python manage.py collectstatic --noinput`

7. Откройте сайт в браузере: `http://localhost:8000`
Админка: `http://localhost:8000/admin`

## Как пользоваться

- На главной странице можно ознакомиться с описанием ресторана, услугами, контактами и отправить сообщение через форму обратной связи.
- На странице "О ресторане" – история, миссия и команда.
- Для бронирования необходимо зарегистрироваться или войти. После авторизации доступна форма выбора столика, даты, времени и количества гостей. При успешном бронировании вы перенаправляетесь на страницу подтверждения.
- В личном кабинете отображаются все бронирования пользователя. Активные брони можно изменять (дата, время, гости) или отменять.
- Администратор через админ-панель может управлять столиками, бронированиями, пользователями, а также редактировать контент сайта (тексты, изображения) через модель SiteContent.

## Важные команды для разработки

Остановка контейнеров: `docker-compose down`
Перезапуск web-сервиса: `docker-compose restart web`
Просмотр логов: `docker-compose logs -f web`
Создание миграций после изменения моделей: `docker-compose exec web python manage.py makemigrations`
Применение миграций: `docker-compose exec web python manage.py migrate`
Загрузка статики после изменения CSS/JS: `docker-compose exec web python manage.py collectstatic --noinput --clear`

## Примечания

- По умолчанию PostgreSQL внутри контейнера доступен на порту 5433 (маппинг 5433:5432), чтобы не конфликтовать с локальным PostgreSQL.
- Для смены фонового изображения или добавления галереи используйте модель SiteContent через админку, загружая изображения в поле `image`.
- Проект полностью соответствует требованиям ТЗ: главная страница, страница "О ресторане", бронирование с формой и проверкой доступности, личный кабинет с регистрацией/авторизацией и управлением бронями, админка, PostgreSQL, Bootstrap, Docker, README.
